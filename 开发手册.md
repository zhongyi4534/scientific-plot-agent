## 2. 两个核心数据结构

### 2.1 PlotSpec —— 所有线都要理解

PlotSpec 是一个**扁平 dict**，是三条线之间传递绘图意图的唯一载体。

**必填字段（缺少任何一个系统会回问用户，不会渲染）：**

| 字段 | 类型 | 合法值 | 示例 |
|------|------|--------|------|
| `chart_type` | str | `"bar"` `"line"` `"scatter"` `"box"` `"heatmap"` | `"bar"` |
| `data_source` | str | DataLoader 返回的缓存 key | `"cache://a1b2c3d4"` |
| `data_x` | **str** | **列名字符串**（不是列的值，只能是单列） | `"method"` |
| `data_y` | **str 或 list[str]** | **列名字符串或列名列表**（不是列的值） | `"accuracy"` 或 `["accuracy", "F1"]` |
| `style_theme` | str | `"normal"` `"morandi"` `"macaron"` `"bright"` `"rococo"` `"earth"` `"science"` `"nature"` | `"nature"` |

> **⚠️ 高频错误：`data_x` / `data_y` 填的必须是列名，不是列的值。**
>
> | 写法 | 是否合法 | 说明 |
> |------|----------|------|
> | `"data_y": "accuracy"` | ✅ | 单列名字符串 |
> | `"data_y": ["accuracy", "F1"]` | ✅ | 多列名字符串列表，用于多指标对比 |
> | `"data_y": [93.5, 94.8, 91.0]` | ❌ | 列的数值，不是列名 |
> | `"data_y": ["accuracy", 93.5]` | ❌ | 混合了列名和数值 |
> | `"data_x": ["method", "dataset"]` | ❌ | X轴只能是单列，不能是列表 |

**可选字段（system/merger.py 会自动填入默认值，不填也不影响校验）：**

| 字段 | 默认值 | 含义 | 适用图表 |
|------|--------|------|----------|
| `data_group_by` | `null` | 分组列名，用于多系列 | bar, line |
| `data_error` | `null` | 误差棒列名 | bar, line |
| `data_filter` | `null` | 行过滤条件（pandas query 语法） | 全部 |
| `label_title` | `""` | 图表标题 | 全部 |
| `label_x` | `""` | X轴标签 | 全部 |
| `label_y` | `""` | Y轴标签 | 全部 |
| `axes_y_scale` | `"linear"` | Y轴缩放：`"linear"` / `"log"`（水平柱状图自动映射到 X 轴） | 全部（heatmap 除外） |
| `axes_y_min` | `null` | Y轴最小值 | 全部 |
| `axes_y_max` | `null` | Y轴最大值 | 全部 |
| `axes_x_tick_rotation` | `0` | X轴刻度旋转角度（度）；非零时直接应用，0 时由渲染层自动检测 | 全部 |
| `axes_x_rotate_labels` | `false` | 标签拥挤时的处理方式：`true`=旋转（30°–45°），`false`=缩小字号 | 全部 |
| `legend_loc` | `null` | 图例位置：`null`/`"auto"`=自动；`"inside"`=图内最优；`"outside_right"`=图外右侧；`"none"`=不显示 | 全部 |
| `style_palette_override` | `null` | 配色覆盖 `"morandi"` `"nature_d"` `"tab10"` `"coolwarm"` | 全部 |
| `style_grid` | `null` | 是否显示网格（`null`=用主题默认） | 全部 |
| `style_line_width` | `null` | 线宽（磅）覆写 | 全部 |
| `style_font_size` | `null` | 基准字号（磅）覆写 | 全部 |
| `style_hatch` | `null` | 柱子纹理；单个字符串如 `"/"` 或列表如 `["/","\\","\|"]`（多分组时轮换）；`null`=不使用 | **bar** |
| `style_edgecolor` | `null` | 柱子/纹理边框颜色，如 `"white"` `"black"`；`null`=用主题默认 | **bar** |
| `style_hatch_linewidth` | `null` | 纹理线宽；`null`=主题默认 0.5；仅 `style_hatch` 不为 `null` 时生效 | **bar** |
| `style_dpi` | `null` | 输出分辨率（DPI）覆写 | 全部 |
| `style_legend_frameon` | `null` | 图例是否有边框（`null`=用主题默认） | 全部 |
| `style_bg_color` | `null` | 背景色覆写，如 `"#1e1e2e"` | 全部 |
| `style_text_color` | `null` | 文字/刻度/轴脊颜色覆写 | 全部 |
| `style_aspect_ratio` | `null` | 宽高比覆写，如 `0.75`（高度 = 宽度 × 该值） | 全部 |
| `style_figure_width` | `null` | 最小图幅宽度下限（英寸）；LayoutEngine 仍可按数据继续扩大 | 全部 |

| `params_alpha` | `0.8` | 点透明度 | **scatter** |

**合法 PlotSpec 示例1：按数据集分组的单指标柱状图**

```json
  "axes_y_max": 100,
  "axes_x_tick_rotation": 30,
  "params_sort": "desc",
  "params_show_values": true

```

**合法 PlotSpec 示例2：多指标并排柱状图（data_y 为列名列表）**

```json
{
  "chart_type": "bar",
  "data_source": "cache://a1b2c3d4",
  "data_x": "method",
  "data_y": ["accuracy", "std"],
  "style_theme": "nature",
```

### 2.2 DataContext —— A线需要，B线生成

DataContext 是 `tools/loader.py` 把 CSV 解析后生成的**可读摘要字符串**，用途是注入到模型 prompt。

**格式规范（固定，A线训练数据的 instruction 里必须用这个格式）：**

```
数据摘要：
- 形状：24行 × 5列
- 列信息：
  · method（类别型，唯一值6个）：BERT-base, RoBERTa-base, XLNet-base, ALBERT-base
  · dataset（类别型，唯一值4个）：SST-2, MR, CR, CoLA
  · accuracy（数值型，范围55.1~95.1）
  · std（数值型，范围0.3~1.1）
- 前2行预览：[["BERT-base","SST-2",93.5,0.3],["BERT-base","MR",87.3,0.4]]
- 缓存key：cache://a1b2c3d4
```

> **注意（Plan B 专用）**：`data_source` 必填字段的值是摘要末尾的缓存 key（`cache://xxxxxxxx`），
> Plan B 路径下 DeepSeek 需从 DataContext 提取该 key 并填入 `data_source`。
>
> **Plan A（微调模型）路径下不同**：fine-tuned 小模型不需要输出 `data_source`，
> 系统会在调用 `generate_spec()` 之后由 `agent.py` 自动注入（见 3.4 节）。

用 `data/example_bar.csv` 实际调用 `load_data()` 可以看到真实输出：

```bash
python -c "from tools.loader import load_data; ctx,_ = load_data('data/example_bar.csv'); print(ctx)"
```

## 3. A线：模型推理

**负责文件：`model/generator.py`（唯一文件）**  
**不能动的文件：除 `model/generator.py` 之外的所有文件**

### 3.1 当前实现：Plan B（DeepSeek API）

`generator.py` 当前实现通过 DeepSeek API（OpenAI 兼容接口）完成推理，已可正常运行。

**运行前提**：在项目根目录创建 `.env` 文件：

```
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
```

程序启动时自动加载，无需手动设置环境变量。

**调试开关**：`generator.py` 顶部有 `DEBUG = True`，开启时每次 API 调用会在终端打印：

```
==================================================
[DeepSeek 首轮] 用户输入: 画柱状图对比各模型准确率
[DeepSeek 原始响应]:
{"chart_type":"bar","data_source":"cache://a1b2c3d4",...}
==================================================
```

关闭调试只需将 `DEBUG = False`。

### 3.2 函数签名（不得修改）

```python
def generate_spec(
    user_input: str,
    data_context: str,
    current_spec: dict | None = None,
) -> dict:
    # ← 只替换函数体，签名一行都不能改
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `user_input` | str | 用户原始自然语言，如"画个柱状图对比各模型准确率" |
| `data_context` | str | DataContext 摘要字符串（格式见 [2.2节](#22-datacontext--a线需要b线生成)） |
| `current_spec` | dict 或 None | 首轮为 `None`；修改轮传入当前完整 PlotSpec |

**返回值约定：**

- `current_spec is None`（首轮）：返回包含所有 5 个必填字段的完整 PlotSpec dict
- `current_spec is not None`（修改轮）：只返回变更字段的 delta dict，例如仅 `{"style_theme": "science"}`
- 返回的必须是 **dict**，不能是 JSON 字符串

### 3.3 多轮对话机制

当前实现**不维护对话历史**，采用"状态式"方案：

- 每次 API 调用只发送两条消息：system prompt + 当前用户输入
- 修改轮时，`current_spec`（当前完整 PlotSpec）作为上下文注入 prompt
- LLM 每次看到的是"当前图表的完整状态"，而非对话历史

这种方案 token 消耗固定（不随轮数增长），适合"明确字段修改"类需求（换风格、改坐标轴等）。若需要支持"改回原来那个颜色"等历史感知需求，可在 `agent.py` 中维护 `message_history` 列表并传入 `generate_spec`，接口签名不需要改变。

### 3.4 训练数据合成流水线

Plan B 完成验证后，A线的最终目标是用本地 **Qwen3-1.7B LoRA 微调模型**替换 Plan B。
替换时只替换 `generate_spec()` 函数体，其余文件零改动。

#### 训练数据格式（Qwen3 ChatML messages）

每条训练样本为**三段** messages，系统提示词**内嵌**在每条记录中：

```json
{
  "messages": [
    {
      "role": "system",
      "content": "<SYSTEM_FIRST_FINETUNE 或 SYSTEM_DELTA_FINETUNE>"
    },
    {
      "role": "user",
      "content": "<DataContext字符串>\n\n用户需求：<user_input>\n输出：/no_think"
    },
    {
      "role": "assistant",
      "content": "{\"chart_type\":\"bar\",\"data_x\":\"method\",\"data_y\":\"accuracy\",\"style_theme\":\"nature\"}"
    }
  ]
}
```

系统提示词按记录类型（`record_type`）动态选择：
- 首轮记录（`record_type="first"`）→ `SYSTEM_FIRST_FINETUNE`（输出完整 PlotSpec）
- 修改轮记录（`record_type="delta"`）→ `SYSTEM_DELTA_FINETUNE`（输出仅含变更字段的 delta dict）

**必须内嵌**而非统一注入，原因是首轮和修改轮的系统提示词不同，训练框架无法在运行时区分。提示词定义在 `model/prompts.py`，修改后需重新运行 `pack_finetune.py` 才能生效。

**MS-SWIFT 训练命令（系统提示词已内嵌，不需要 `--system` 参数）：**
```bash
swift sft \
  --model Qwen/Qwen3-1.5B-Instruct \
  --dataset data/finetune/train.jsonl \
  --val_dataset data/finetune/val.jsonl \
  --train_type lora \
  ...
```

**与 Plan B 的关键差异：**

| | Plan B（DeepSeek API） | Plan A（微调模型） |
|--|--|--|
| system prompt | `_SYSTEM_FIRST`（含 data_source 必填） | `SYSTEM_FIRST_FINETUNE`（不含 data_source，训练时由框架注入） |
| assistant 输出 | 含 `data_source` 字段 | 不含 `data_source`，由系统注入 |
| user message | 含 `_FEW_SHOT` 示例 | 无 few-shot，模型从微调权重中学习 |
| `/no_think` | 不需要 | user message 末尾必须加，防止 Qwen3 产生空推理链 |

`format_user_message()` 定义在 `model/prompts.py`，
训练打包（`scripts/pack_finetune.py`）和 A线推理时必须使用同一个函数。

#### 一键合成流水线

```bash
# 快速测试（2个CSV，跳过渲染校验，修改轮只用规则驱动）
python scripts/run_synthesis.py --limit 2 --no-render --delta-no-llm

# 完整运行
python scripts/run_synthesis.py --model deepseek-chat

# 跳过 CSV 生成（data/train/ 已存在时）
python scripts/run_synthesis.py --skip-csv --model deepseek-chat

# 只重新生成修改轮并打包（首轮已校验完成时）
python scripts/run_synthesis.py --skip-to-delta
```

五步依次执行，每步可单独调用：

| 步骤 | 脚本 | 输入 | 输出 |
|------|------|------|------|
| 1 | `gen_csv.py` | — | `data/train/*.csv`（64 个） |
| 2 | `gen_pairs.py --model <name>` | `data/train/*.csv` | `data/pairs/raw_pairs.jsonl` |
| 3 | `validate_pairs.py` | `raw_pairs.jsonl` | `valid_pairs.jsonl` + `reject_log.jsonl` |
| 4 | `gen_delta.py --model <name>` | `valid_pairs.jsonl` | `data/pairs/delta_pairs.jsonl` |
| 5 | `pack_finetune.py` | `valid_pairs.jsonl` + `delta_pairs.jsonl` | `data/finetune/train.jsonl` + `val.jsonl` |

**步骤2（gen_pairs.py）细节：**
- 调用 DeepSeek API（OpenAI 兼容接口），每个 CSV 合成 5 条多样化首轮配对
- 单独调用时默认模型为 `deepseek-v4-pro`；通过 `run_synthesis.py` 调用时继承 `--model` 参数（默认 `deepseek-chat`）
- synthesis prompt 从 `model/prompts.py` 动态读取 schema 约束，不硬编码枚举值
- 多样性维度：图表类型 × 风格主题 × 语气风格 × 可选字段使用
- 所有 CSV 的列名已做随机化（`gen_csv.py` 末尾的 `_apply_column_aliases()`），
  如 `model→method`、`accuracy→acc`、`val_loss→dev_loss`，增加训练覆盖范围

**步骤3（validate_pairs.py）四级校验：**

1. `fill_defaults()` + `validator.py`：必填字段完整性 + 枚举值合法性
2. 列名存在性：`data_x`、`data_y`、`data_group_by` 等引用的列必须在对应 CSV 中存在
3. 语义合理性：字段组合约束（如 scatter 用类别型 X 不能加回归线、box 图 data_y 不能是列表）
4. `render_plot()`：实际渲染出图，验证后自动删除 PNG（可用 `--no-render` 跳过）

**步骤4（gen_delta.py）细节：**

从 `valid_pairs.jsonl` 的首轮记录出发，为每条记录生成若干修改轮 `(user_input, delta)` 配对，写入 `delta_pairs.jsonl`。支持两种策略（可混用）：
- **规则驱动**（`--no-llm`，快速）：随机挑选简单字段变更（换主题、调坐标轴范围等），按模板拼接中文请求
- **LLM 生成**（默认，多样）：调用 DeepSeek API，给定当前 spec 后让模型想象各种合理修改需求

delta 写出前同样经过 `fill_defaults + validate + merge_delta` 校验，确保合并后的 spec 合法。

### 4.2 themes.py —— 静态主题参数（已完整实现）

#### 设计原则：Theme 只管"风格"，不管"尺寸"

`ThemeConfig` 存储**与数据无关**的静态风格属性，所有字段在主题注册时写死，渲染过程中不会被覆盖：

| 字段 | 类型 | 作用 |
|------|------|------|
| `font_family` | str | 字体族（Arial / Times New Roman / DejaVu Sans） |
| `font_size` | int | **基准字号**（磅），实际字号由 LayoutEngine 按图幅缩放后使用 |
| `line_width` | float | 线宽基准（磅） |
| `figure_width` | float | **最小基准图幅宽度**（英寸），LayoutEngine 只扩大不缩小 |
| `aspect_ratio` | float | 宽高比（高度 = 宽度 × aspect_ratio），如 `0.75` 表示 4:3 |
| `dpi` | int | 输出分辨率 |
| `spines` | list[str] | 保留的轴脊方向，如 `["left", "bottom"]` |
| `grid` | bool | 是否显示网格 |
| `grid_style` | str | 网格线型，如 `"--"` |
| `legend_frameon` | bool | 图例是否有边框 |
| `palette` | list[str] | 配色列表（十六进制） |
| `bg_color` | str | 背景色（默认 `"white"`；`earth` 主题为米黄 `"#FDF6E3"`） |
| `text_color` | str | 文字/刻度/轴脊颜色（默认 `"black"`） |
| `hatch` | list[str] \| None | 柱子纹理序列（默认 `None`=不使用）；多分组时按索引轮换；仅 bar 图生效 |
| `edgecolor` | str \| None | 柱子/纹理边框颜色（默认 `None`=matplotlib默认）；仅 bar 图生效 |
| `hatch_linewidth` | float | 纹理线宽（默认 `0.5`）；仅 `hatch` 不为 None 时生效 |

> **注意**：`ThemeConfig` 中**不存在** `figure_height` 和 `legend_fontsize` 字段。
> 高度由 `figure_width × aspect_ratio` 推导；图例字号由 LayoutEngine 以 `font_size × 0.85` 推导。
> 这两个值是计算结果，不是配置项。
>
> **`style_figure_width` 的语义**：覆写后作为新的最小宽度下限，LayoutEngine 的 `max(theme.figure_width, ...)` 逻辑保持不变，数据密集时仍可继续扩大。

八个预设主题的视觉特征：

| key | 视觉特征 | 字体 | figure_width | aspect_ratio |
|-----|---------|------|:---:|:---:|
| `normal` | 简洁常规、柔和暖色 | Arial | 3.5" | 0.75 |
| `morandi` | 低饱和莫兰迪、带交错纹理 | Arial | 6.0" | 0.667 |
| `macaron` | 马卡龙轻快色调、带网格与图例框 | Arial | 5.0" | 0.8 |
| `bright` | 高对比鲜艳、Times 衬线字体 | Times New Roman | 6.0" | 0.75 |
| `rococo` | 洛可可低对比淡色、带图例框 | Arial | 5.5" | 0.75 |
| `earth` | 深沉大地色、米黄背景 | Times New Roman | 5.0" | 0.75 |
| `science` | Science 期刊冷蓝色调 | Times New Roman | 5.0" | 0.75 |
| `nature` | Nature 期刊规范、细线小字 | Arial | 4.0" | 0.75 |

**中文字体**：渲染器在创建 Figure 前统一将 `Microsoft YaHei` 设为首选字体，保证中文标题、轴标签正常显示；主题的 `font_family` 作为备选（西文字符优先使用主题字体）。

不允许新增或删除主题名称；如确实需要，需同步修改 `schema.py` 里的 `STYLE_THEMES`（高风险操作，改前在群里说一声）。具体操作见 [8.2 新增预设风格](#82-新增预设风格主题)。

#### 主题覆写机制

用户可以通过 spec 里的 `style_*` 字段在不切换主题的前提下覆写任意 ThemeConfig 属性。渲染器在 `apply_theme()` 之后调用 `apply_style_overrides(theme, spec)`，对所有非 `null` 的 `style_*` 字段用 `dataclasses.replace()` 生成新 ThemeConfig 实例，原注册表不受影响。覆写优先级：`style_*` spec 字段 > 主题默认值。

### 4.3 layout.py —— 动态布局引擎（已完整实现）

#### 设计原则：LayoutEngine 只管"尺寸"，不管"风格"

`LayoutEngine`（`tools/layout.py`）在每次渲染时根据**实际数据和 PlotSpec** 动态计算布局参数，结果存入 `LayoutParams` dataclass，再由渲染器使用。所有计算均有 try/except 保底，失败时自动退回 theme 基准值。

#### LayoutParams 字段说明

```python
@dataclass
class LayoutParams:
    figure_width: float       # 最终图幅宽度（英寸），可能大于 theme.figure_width
    figure_height: float      # 最终图幅高度 = figure_width × theme.aspect_ratio
    font_size: float          # 最终字号（磅），夹在 [6, 14]
    legend_fontsize: float    # = font_size × 0.85
    tick_rotation: int        # 0 = 渲染层 post-draw 自动检测；非零 = 用户明确指定，直接应用
    tick_ha: str              # 刻度水平对齐："right"（旋转时）或 "center"
    legend_loc: str           # "inside_best" | "outside_right" | "none"
    y_min: float | None       # Y 轴下限（合并了用户指定值与自动收紧值）
    y_max: float | None       # Y 轴上限（来自 spec.axes_y_max）
    bar_width: float          # 单根柱子/箱体宽度
    marker_size: float        # line: markersize 直径(pts)；scatter: s 面积(pts²)
    annot_fontsize: float     # 热力图注释字体大小
    capsize: float            # 误差棒帽宽
    x_max_ticks: int | None   # 折线图 X 轴最多刻度数；None = 不限制
    rotate_labels: bool       # True=间距不足时旋转标签(30°–45°)；False=缩小字号（来自 axes_x_rotate_labels）
```

#### 各图表类型的动态计算逻辑

| 参数 | bar | line | scatter | box | heatmap |
|------|-----|------|---------|-----|---------|
| **figure_width** | `max(theme_w, n_x×n_groups×0.20 + n_x×0.12 + 1.5)` | 等于 `theme_w` | 等于 `theme_w` | `max(theme_w, n_x×0.5 + 1.2)` | `max(theme_w, n_cols×0.5 + 1.5)` |
| **figure_height** | `width × theme.aspect_ratio` | 同左 | 同左 | 同左 | `max(theme_w × aspect, n_rows×0.5 + 1.0)` |
| **font_size** | `clamp(base × (w/base_w)^0.4, 6, 14)` | 同左 | 同左 | 同左 | 同左 |
| **bar_width** | `clamp(0.75/n_groups, 0.08, 0.4)` | — | — | `clamp(0.6/n_x, 0.1, 0.5)` | — |
| **tick_rotation** | 用户指定非零→直接用；0→post-draw自动测量 | 同左 | 同左 | 同左 | 同左 |
| **rotate_labels** | 来自 `axes_x_rotate_labels`（默认 false） | 同左 | 同左 | 同左 | 同左 |
| **legend_loc** | n_groups≤5: inside; >5: outside | n_series≤1: none; ≤2: inside; ≥3: outside | n_series≤0: none; ≤5: inside; >5: outside | none | none |
| **y_min** | 用户指定 or 自动收紧 | 同左 | 同左 | 同左 | None |
| **marker_size** | — | n>50:3; n>20:4; 否则:5（直径） | `clamp(60/√n, 10, 80)`（面积） | — | — |
| **annot_fontsize** | — | — | — | — | n_cells>20: 6; 否则: font×0.85 |
| **x_max_ticks** | — | n>20: 10 | — | — | — |

> **Y轴自动收紧规则**：若数据最小值 > 最大值的 50%（数据远离 0），则自动将 Y 轴下限设为 `data_min - span×15%` 并取整到量级，否则保持 None（让 matplotlib 从 0 开始）。用户显式设置 `axes_y_min` 时，该自动计算被跳过。

> **图例移外规则**：当图例条目数 > 5 时，`legend_loc = "outside_right"`，同时 `figure_width` 额外加 1.2 英寸为图例留白。

> **刻度旋转规则（draw-measure-fix）**：`tick_rotation=0` 时，渲染器在绘图完成后调用 `fig.canvas.draw()`，用 `get_window_extent()` 获取每个标签的实际像素 bbox，若相邻标签间距 < 安全间距（默认 3pt），则触发修正：
> - `axes_x_rotate_labels=true`：用 `arccos((slot_px − safety_px) / max_w_px)` 计算最小所需旋转角，夹在 [30°, 45°]
> - `axes_x_rotate_labels=false`（默认）：按比例缩小字号，最小 3pt

#### 调用方式

渲染器不需要直接实例化 `LayoutParams`，只需调用：

```python
from tools.layout import compute_layout

layout = compute_layout(df, spec, theme)
# layout.figure_width, layout.font_size, layout.legend_loc ... 等字段随即可用
```

**渲染流水线：**

```
render_plot(spec, data_source)
  ├─ _resolve_dataframe()     从缓存或文件路径取回 DataFrame
  ├─ apply_theme()            从 themes.py 取 ThemeConfig（静态风格）
  ├─ apply_style_overrides()  将 spec 中的 style_* 字段覆写到 ThemeConfig（若全为 null 则跳过）
  ├─ compute_layout()         从 layout.py 计算 LayoutParams（动态布局）
  └─ RENDERERS[chart_type]()  调用对应子渲染器
       ├─ _prepare_axes()        创建 Figure/Axes，figsize = layout.figure_width/height
       ├─ ... 绘制数据 ...
       ├─ _apply_axis_limits()   应用 Y 轴范围（来自 layout，已合并用户值与自动收紧）
       └─ _apply_theme_to_fig()  统一处理：字号/轴脊/网格/刻度旋转/图例位置
```

**`_apply_theme_to_fig()` 统一处理三件事**（子渲染器无需自行设置）：

1. **刻度旋转**：`layout.tick_rotation != 0` 时直接调用 `ax.tick_params(labelrotation=...)`；为 0 时调用 `_auto_rotate_xlabels(fig, ax, rotate=layout.rotate_labels)` 进行 post-draw 自动检测（旋转或缩字，取决于 `rotate_labels`）
2. **图例**：根据 `layout.legend_loc` 三态创建图例：
   - `"inside_best"` → `ax.legend(loc="best")`
   - `"outside_right"` → `ax.legend(bbox_to_anchor=(1.02, 1))`（图外右侧）
   - `"none"` → 不创建（heatmap / 单系列图的默认值）
3. **样式**：字号 / 轴脊可见性与线宽 / 网格 / 背景色（dark 主题）

**各子渲染器读取的关键 spec 字段：**

**bar 图：**
- `data_y` 为字符串时：单指标，支持 `data_group_by`、`data_error`、`params_sort`、`params_stacked`
- `data_y` 为列表时：自动 melt 宽表，每列名作为分组维度，`data_group_by` 被忽略
- 支持 `params_orientation`（横/纵）、`params_show_values`；纹理通过 `style_hatch` 覆写（单字符串或列表）
- `params_sort` 对无分组图按 Y 值排序；对分组图按各分组 Y 均值排序

**line 图：**
- `data_y` 为字符串时：单条线，支持 `data_group_by`（多条线）、`data_error`（误差带）
- `data_y` 为列表时：每列对应一条折线，直接循环绘制
- 支持 `params_smooth`、`params_linestyle`、`params_line_colors`、`params_show_markers`

**scatter / box / heatmap：** 详见代码注释及 2.1 节字段表。

**heatmap 特殊说明：**

长表路径（标准）：`data_x` = 列轴列名，`data_y` = 行轴列名，`params_heatmap_value` = 热力值列（null=自动取第一个非轴数值列）。渲染器内部调用 `df.pivot(index=data_y, columns=data_x, values=value_col)`。

宽表路径（自动识别）：若 `data_x` 的值不是 DataFrame 中的列名，渲染器判定为宽表矩阵格式，执行 `df.set_index(data_y).select_dtypes(include="number")` 直接构造热力矩阵，无需 pivot。LLM 此时应将 `data_x` 填为列轴的概念名（任意字符串，如 `"dataset"`），`data_y` 填行标签列名。

`data_filter`（pandas query 过滤）**当前仅在 heatmap 渲染器中生效**，其他图表类型的 `_render_*` 函数未调用该字段，填写后不会过滤数据。如需为其他图表类型启用，在对应 `_render_xxx()` 函数开头加一行 `if filter_expr := spec.get("data_filter"): df = df.query(filter_expr)` 即可。

#### 第二步：`tools/layout.py` —— 新增布局计算函数

在 `_DISPATCHERS` 字典前添加新图表的布局函数，并注册：

```python
def _layout_violin(df: pd.DataFrame, spec: dict, theme: ThemeConfig) -> LayoutParams:
    x_col = spec.get("data_x", "")
    y_col = spec.get("data_y", "")
    n_cat = int(df[x_col].nunique()) if x_col in df.columns else 4

    min_width = n_cat * 0.6 + 1.2
    fig_w = max(theme.figure_width, min_width)
    fig_h = fig_w * theme.aspect_ratio
    bar_width = max(0.1, min(0.6, 0.7 / max(n_cat, 1)))
    font_size = _scale_font(theme.font_size, fig_w, theme.figure_width)
    y_min, y_max = _compute_y_range(df, spec, y_col)
    user_rot = spec.get("axes_x_tick_rotation")
    if user_rot:
        tick_rotation = int(user_rot)
        tick_ha = "right" if tick_rotation > 0 else "center"
    else:
        tick_rotation, tick_ha = 0, "center"
    rotate_labels = bool(spec.get("axes_x_rotate_labels", False))
    return LayoutParams(
        figure_width=fig_w, figure_height=fig_h,
        font_size=font_size, legend_fontsize=round(font_size * 0.85, 1),
        tick_rotation=tick_rotation, tick_ha=tick_ha,
        legend_loc="none",
        y_min=y_min, y_max=y_max,
        bar_width=bar_width, marker_size=5.0,
        annot_fontsize=font_size, capsize=float(theme.line_width) * 2,
        x_max_ticks=None,
        rotate_labels=rotate_labels,
    )

_DISPATCHERS = {
    ...
    "violin": _layout_violin,   # ← 追加
}
```

#### 第三步：`tools/renderer.py` —— 实现渲染函数并注册

子渲染器签名必须包含 `layout: LayoutParams` 参数，并通过 `_apply_theme_to_fig()` 统一处理图例和刻度旋转：

```python
def _render_violin(
    df: pd.DataFrame, spec: dict, theme: ThemeConfig, layout: LayoutParams
) -> plt.Figure:
    fig, ax = _prepare_axes(spec, theme, layout)
    sns.violinplot(
        data=df, x=spec["data_x"], y=spec["data_y"],
        hue=spec.get("data_group_by"),
        inner=spec.get("params_violin_inner", "box"),
        palette=theme.palette, linewidth=theme.line_width,
        width=layout.bar_width, ax=ax,
    )
    _apply_axis_limits(ax, layout)
    _apply_theme_to_fig(fig, ax, theme, layout)
    return fig

RENDERERS = {
    ...
    "violin": _render_violin,   # ← 追加
}
```

### 8.2 新增预设风格（主题）

以新增 ACL 会议风格（`"acl"`）为例，需要改动 **2 个文件**。

#### 第一步：`schema.py`

```python
STYLE_THEMES = ["normal", "morandi", "macaron", "bright", "rococo", "earth", "science", "nature", "acl"]   # ← 追加
```

#### 第二步：`tools/themes.py`

`ThemeConfig` 只包含静态风格字段，不含 `figure_height`（由 `aspect_ratio` 推导）和 `legend_fontsize`（由 LayoutEngine 推导）：

```python
THEMES: dict[str, ThemeConfig] = {
    ...
    "acl": ThemeConfig(
        font_family="Times New Roman",
        font_size=10,           # 基准字号（标准图幅下），实际字号由 LayoutEngine 缩放
        line_width=0.8,
        figure_width=6.0,       # 最小基准宽度（英寸），LayoutEngine 只扩大不缩小
        aspect_ratio=0.625,     # 高度 = 6.0 × 0.625 = 3.75 英寸
        dpi=300,
        spines=["left", "bottom"],
        grid=False,
        grid_style="--",
        legend_frameon=True,
        palette=["#2196F3", "#F44336", "#4CAF50", "#FF9800", "#9C27B0"],
    ),
}
```

若新主题需要深色背景或特殊背景色，加上 `bg_color` 和 `text_color` 字段（参考 `earth` 主题的 `bg_color="#FDF6E3"`）。

**`aspect_ratio` 换算参考：**

| 期望图幅 | aspect_ratio |
|---------|:---:|
| 3.5" × 2.625" | 0.75 |
| 5.0" × 3.75" | 0.75 |
| 6.0" × 4.0" | 0.667 |
| 6.0" × 3.75" | 0.625 |
| 7.0" × 4.5" | 0.643 |
