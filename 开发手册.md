## 2. 两个核心数据结构

### 2.1 PlotSpec —— 所有线都要理解

PlotSpec 是一个**扁平 dict**，是三条线之间传递绘图意图的唯一载体。

**必填字段（缺少任何一个系统会回问用户，不会渲染）：**

| 字段 | 类型 | 合法值 | 示例 |
|------|------|--------|------|
| `chart_type` | str | `"bar"` `"line"` `"scatter"` `"box"` `"heatmap"` | `"bar"` |
| `data_source` | str | DataLoader 返回的缓存 key | `"cache://a1b2c3d4"` |
| `data_x` | **str** | **列名字符串**（不是列的值，只能是单列） | `"method"` |
| `data_y` | **str 或 list[str]** | **列名字符串或列名列表**（不是列的值） | `"accuracy"` 或 `["accuracy", "F1"]` |
| `style_theme` | str | `"clean"` `"vivid"` `"nature"` `"ieee"` `"morandi"` `"dark"` | `"nature"` |

> **⚠️ 高频错误：`data_x` / `data_y` 填的必须是列名，不是列的值。**
>
> | 写法 | 是否合法 | 说明 |
> |------|----------|------|
> | `"data_y": "accuracy"` | ✅ | 单列名字符串 |
> | `"data_y": ["accuracy", "F1"]` | ✅ | 多列名字符串列表，用于多指标对比 |
> | `"data_y": [93.5, 94.8, 91.0]` | ❌ | 列的数值，不是列名 |
> | `"data_y": ["accuracy", 93.5]` | ❌ | 混合了列名和数值 |
> | `"data_x": ["method", "dataset"]` | ❌ | X轴只能是单列，不能是列表 |

**可选字段（system/merger.py 会自动填入默认值，不填也不影响校验）：**

| 字段 | 默认值 | 含义 | 适用图表 |
|------|--------|------|----------|
| `data_group_by` | `null` | 分组列名，用于多系列 | bar, line |
| `data_error` | `null` | 误差棒列名 | bar, line |
| `data_filter` | `null` | 行过滤条件（pandas query 语法） | 全部 |
| `label_title` | `""` | 图表标题 | 全部 |
| `label_x` | `""` | X轴标签 | 全部 |
| `label_y` | `""` | Y轴标签 | 全部 |
| `axes_x_scale` | `"linear"` | X轴刻度类型 `"linear"` / `"log"` | 全部 |
| `axes_y_scale` | `"linear"` | Y轴刻度类型 | 全部 |
| `axes_y_min` | `null` | Y轴最小值 | 全部 |
| `axes_y_max` | `null` | Y轴最大值 | 全部 |
| `axes_x_tick_rotation` | `0` | X轴刻度旋转角度（度） | 全部 |
| `style_palette_override` | `null` | 配色覆盖 `"morandi"` `"nature_d"` `"tab10"` `"coolwarm"` | 全部 |
| `params_orientation` | `"vertical"` | 柱方向 `"vertical"` / `"horizontal"` | **bar** |
| `params_stacked` | `false` | 是否堆叠 | **bar** |
| `params_sort` | `null` | 排序 `"asc"` / `"desc"` / `null` | **bar** |
| `params_show_values` | `false` | 柱顶显示数值 | **bar** |
| `params_hatch` | `null` | 柱子纹理，如 `"/"` `"\\"` `"\|"` `"-"` `"+"` `"x"`；`null`=不使用 | **bar** |
| `params_edgecolor` | `null` | 柱子/纹理边框颜色，如 `"white"` `"black"`；`null`=matplotlib默认 | **bar** |
| `params_hatch_linewidth` | `0.5` | 纹理线宽；仅 `params_hatch` 不为 `null` 时生效 | **bar** |
| `params_markers` | `true` | 显示数据点标记 | **line** |
| `params_smooth` | `false` | 曲线平滑 | **line** |
| `params_linestyle` | `"solid"` | 线型：`"solid"` / `"dashed"` / `"dotted"` / `"dashdot"` | **line** |
| `params_line_colors` | `null` | 按线顺序的颜色列表，覆盖主题配色；`null`=使用主题配色 | **line** |
| `params_marker_style` | `null` | 标记样式，如 `"o"` `"s"` `"^"` `"D"`；`null`=使用 `"o"` | **line, scatter** |
| `params_marker_size` | `4` | 标记大小；line 用 `markersize`，scatter 内部转换为面积 `msize**2` | **line, scatter** |
| `params_alpha` | `0.8` | 点透明度 | **scatter** |
| `params_show_regression` | `false` | 显示回归线 | **scatter** |
| `params_show_points` | `"outliers"` | 显示数据点策略 `"all"` / `"outliers"` / `"none"` | **box** |
| `params_notch` | `false` | 缺口箱线图 | **box** |
| `params_annot` | `true` | 单元格数值标注 | **heatmap** |
| `params_fmt` | `".2f"` | 数值格式字符串 | **heatmap** |

**合法 PlotSpec 示例1：按数据集分组的单指标柱状图**

```json
{
  "chart_type": "bar",
  "data_source": "cache://a1b2c3d4",
  "data_x": "method",
  "data_y": "accuracy",
  "style_theme": "nature",
  "data_group_by": "dataset",
  "data_error": "std",
  "label_title": "模型准确率对比",
  "label_x": "模型",
  "label_y": "准确率 (%)",
  "axes_y_min": 50,
  "axes_y_max": 100,
  "axes_x_tick_rotation": 30,
  "params_sort": "desc",
  "params_show_values": true
}
```

**合法 PlotSpec 示例2：多指标并排柱状图（data_y 为列名列表）**

```json
{
  "chart_type": "bar",
  "data_source": "cache://a1b2c3d4",
  "data_x": "method",
  "data_y": ["accuracy", "std"],
  "style_theme": "nature",
  "label_title": "多指标对比",
  "label_x": "模型",
  "label_y": "值"
}
```

> `data_y` 是列表时，渲染器会自动将宽表 melt 成长表，每列名成为一个分组，
> `data_group_by` 字段在此情况下被忽略。

**合法 PlotSpec 示例3：多条训练曲线（line 图 + data_y 列表）**

```json
{
  "chart_type": "line",
  "data_source": "cache://b2c3d4e5",
  "data_x": "step",
  "data_y": ["train_loss", "val_loss"],
  "style_theme": "clean",
  "label_title": "训练与验证损失",
  "label_x": "训练步数",
  "label_y": "Loss"
}
```

### 2.2 DataContext —— A线需要，B线生成

DataContext 是 `tools/loader.py` 把 CSV 解析后生成的**可读摘要字符串**，用途是注入到模型 prompt。

**格式规范（固定，A线训练数据的 instruction 里必须用这个格式）：**

```
数据摘要：
- 形状：24行 × 5列
- 列信息：
  · method（类别型，唯一值6个）：BERT-base, RoBERTa-base, XLNet-base, ALBERT-base
  · dataset（类别型，唯一值4个）：SST-2, MR, CR, CoLA
  · accuracy（数值型，范围55.1~95.1）
  · std（数值型，范围0.3~1.1）
- 前2行预览：[["BERT-base","SST-2",93.5,0.3],["BERT-base","MR",87.3,0.4]]
- 缓存key：cache://a1b2c3d4
```

> **注意**：`data_source` 必填字段的值就是摘要末尾这行的缓存 key（`cache://xxxxxxxx`），
> 而不是 CSV 文件路径。LLM 收到 DataContext 后应提取该 key 填入 `data_source`。

用 `data/example_bar.csv` 实际调用 `load_data()` 可以看到真实输出：

```bash
python -c "from tools.loader import load_data; ctx,_ = load_data('data/example_bar.csv'); print(ctx)"
```

**负责文件：`model/generator.py`（唯一文件）**

### 3.1 当前实现：Plan B（DeepSeek API）

`generator.py` 当前实现通过 DeepSeek API（OpenAI 兼容接口）完成推理，已可正常运行。

**运行前提**：在项目根目录创建 `.env` 文件：

```

```

程序启动时自动加载，无需手动设置环境变量。

**调试开关**：`generator.py` 顶部有 `DEBUG = True`，开启时每次 API 调用会在终端打印：

```
==================================================
[DeepSeek 首轮] 用户输入: 画柱状图对比各模型准确率
[DeepSeek 原始响应]:
{"chart_type":"bar","data_source":"cache://a1b2c3d4",...}
==================================================
```

关闭调试只需将 `DEBUG = False`。

### 3.2 函数签名（不得修改）

```python
def generate_spec(
    user_input: str,
    data_context: str,
    current_spec: dict | None = None,
) -> dict:
    # ← 只替换函数体，签名一行都不能改
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `user_input` | str | 用户原始自然语言，如"画个柱状图对比各模型准确率" |
| `data_context` | str | DataContext 摘要字符串（格式见 [2.2节](#22-datacontext--a线需要b线生成)） |
| `current_spec` | dict 或 None | 首轮为 `None`；修改轮传入当前完整 PlotSpec |

**返回值约定：**

- `current_spec is None`（首轮）：返回包含所有 5 个必填字段的完整 PlotSpec dict
- `current_spec is not None`（修改轮）：只返回变更字段的 delta dict，例如仅 `{"style_theme": "ieee"}`
- 返回的必须是 **dict**，不能是 JSON 字符串

### 3.3 多轮对话机制

当前实现**不维护对话历史**，采用"状态式"方案：

- 每次 API 调用只发送两条消息：system prompt + 当前用户输入
- 修改轮时，`current_spec`（当前完整 PlotSpec）作为上下文注入 prompt
- LLM 每次看到的是"当前图表的完整状态"，而非对话历史

这种方案 token 消耗固定（不随轮数增长），适合"明确字段修改"类需求（换风格、改坐标轴等）。若需要支持"改回原来那个颜色"等历史感知需求，可在 `agent.py` 中维护 `message_history` 列表并传入 `generate_spec`，接口签名不需要改变。

### 3.4 A线目标：LoRA 微调替换

Plan B 完成验证后，A线的最终目标是用本地 1.5B LoRA 模型替换 Plan B。替换时：
- **只替换 `generate_spec()` 函数体**，其余所有文件零改动
- Plan B 的 prompt 格式（few-shot + DataContext + 用户需求）可直接作为微调数据格式参考

**微调数据格式：**

```json
{
  "instruction": "<DataContext摘要>\n\n用户需求：<user_input>",
  "output": "<合法的PlotSpec JSON字符串>"
}
```

首轮样本（output 包含完整 PlotSpec）：

```json
{
  "instruction": "数据摘要：\n...\n- 缓存key：cache://a1b2c3d4\n\n用户需求：画一张柱状图，比较不同模型的准确率，用nature风格",
  "output": "{\"chart_type\":\"bar\",\"data_source\":\"cache://a1b2c3d4\",\"data_x\":\"method\",\"data_y\":\"accuracy\",\"style_theme\":\"nature\"}"
}
```

修改轮样本（output 只包含变更字段）：

```json
{
  "instruction": "...\n\n当前配置：{...}\n\n修改需求：换成ieee风格",
  "output": "{\"style_theme\":\"ieee\"}"
}
```

**关键注意事项：**

1. `data_x` / `data_y` 强化训练：1.5B 模型的已知失败模式是把列的值填入这两个字段。专项样本里要大量展示"正确 = 列名字符串"的例子。
2. 用 `outlines` 库做 constrained decoding 是兜底手段，保证 JSON 格式合法，但不能保证语义正确。
3. 生成微调数据后，确认 DataContext 格式与 `loader.py` 实际输出一致。

### 4.2 themes.py —— 静态主题参数（已完整实现）

#### 设计原则：Theme 只管"风格"，不管"尺寸"

`ThemeConfig` 存储**与数据无关**的静态风格属性，所有字段在主题注册时写死，渲染过程中不会被覆盖：

| 字段 | 类型 | 作用 |
|------|------|------|
| `font_family` | str | 字体族（Arial / Times New Roman / DejaVu Sans） |
| `font_size` | int | **基准字号**（磅），实际字号由 LayoutEngine 按图幅缩放后使用 |
| `line_width` | float | 线宽基准（磅） |
| `figure_width` | float | **最小基准图幅宽度**（英寸），LayoutEngine 只扩大不缩小 |
| `aspect_ratio` | float | 宽高比（高度 = 宽度 × aspect_ratio），如 `0.75` 表示 4:3 |
| `dpi` | int | 输出分辨率 |
| `spines` | list[str] | 保留的轴脊方向，如 `["left", "bottom"]` |
| `grid` | bool | 是否显示网格 |
| `grid_style` | str | 网格线型，如 `"--"` |
| `legend_frameon` | bool | 图例是否有边框 |
| `palette` | list[str] | 配色列表（十六进制） |
| `bg_color` | str | 背景色（默认 `"white"`，`dark` 主题为 `"#1e1e2e"`） |
| `text_color` | str | 文字/刻度/轴脊颜色（默认 `"black"`，`dark` 主题为 `"#cdd6f4"`） |

> **注意**：`ThemeConfig` 中**不存在** `figure_height` 和 `legend_fontsize` 字段。
> 高度由 `figure_width × aspect_ratio` 推导；图例字号由 LayoutEngine 以 `font_size × 0.85` 推导。
> 这两个值是计算结果，不是配置项。

六个预设主题的视觉特征：

| key | 视觉特征 | 字体 | figure_width | aspect_ratio |
|-----|---------|------|:---:|:---:|
| `clean` | 白底、灰色系、无装饰 | DejaVu Sans | 6.0" | 0.667 |
| `vivid` | 高饱和配色、适合对比展示 | DejaVu Sans | 6.0" | 0.667 |
| `nature` | 细线、小字、Nature 规范图幅 | Arial | 3.5" | 0.75 |
| `ieee` | 网格线、IEEE 规范图幅 | Times New Roman | 3.5" | 0.75 |
| `morandi` | 低饱和、柔和、视觉舒适 | Arial | 5.0" | 0.75 |
| `dark` | 深底浅线、适合演示文稿 | DejaVu Sans | 7.0" | 0.667 |

**中文字体**：渲染器在创建 Figure 前统一将 `Microsoft YaHei` 设为首选字体，保证中文标题、轴标签正常显示；主题的 `font_family` 作为备选（西文字符优先使用主题字体）。

不允许新增或删除主题名称；如确实需要，需同步修改 `schema.py` 里的 `STYLE_THEMES`（高风险操作，改前在群里说一声）。具体操作见 [8.2 新增预设风格](#82-新增预设风格主题)。

### 4.3 layout.py —— 动态布局引擎（已完整实现）

#### 设计原则：LayoutEngine 只管"尺寸"，不管"风格"

`LayoutEngine`（`tools/layout.py`）在每次渲染时根据**实际数据和 PlotSpec** 动态计算布局参数，结果存入 `LayoutParams` dataclass，再由渲染器使用。所有计算均有 try/except 保底，失败时自动退回 theme 基准值。

#### LayoutParams 字段说明

```python
@dataclass
class LayoutParams:
    figure_width: float       # 最终图幅宽度（英寸），可能大于 theme.figure_width
    figure_height: float      # 最终图幅高度 = figure_width × theme.aspect_ratio
    font_size: float          # 最终字号（磅），夹在 [6, 14]
    legend_fontsize: float    # = font_size × 0.85
    tick_rotation: int        # X 轴刻度旋转角度（0 / 45 / 60）
    tick_ha: str              # 刻度水平对齐："right"（旋转时）或 "center"
    legend_loc: str           # "inside_best" | "outside_right" | "none"
    y_min: float | None       # Y 轴下限（合并了用户指定值与自动收紧值）
    y_max: float | None       # Y 轴上限（来自 spec.axes_y_max）
    bar_width: float          # 单根柱子/箱体宽度
    marker_size: float        # line: markersize 直径(pts)；scatter: s 面积(pts²)
    annot_fontsize: float     # 热力图注释字体大小
    capsize: float            # 误差棒帽宽
    x_max_ticks: int | None   # 折线图 X 轴最多刻度数
```

#### 各图表类型的动态计算逻辑

| 参数 | bar | line | scatter | box | heatmap |
|------|-----|------|---------|-----|---------|
| **figure_width** | `max(theme_w, n_x×n_groups×0.20 + n_x×0.12 + 1.5)` | 等于 `theme_w` | 等于 `theme_w` | `max(theme_w, n_x×0.5 + 1.2)` | `max(theme_w, n_cols×0.5 + 1.5)` |
| **figure_height** | `width × theme.aspect_ratio` | 同左 | 同左 | 同左 | `max(theme_w × aspect, n_rows×0.5 + 1.0)` |
| **font_size** | `clamp(base × (w/base_w)^0.4, 6, 14)` | 同左 | 同左 | 同左 | 同左 |
| **bar_width** | `clamp(0.75/n_groups, 0.08, 0.4)` | — | — | `clamp(0.6/n_x, 0.1, 0.5)` | — |
| **tick_rotation** | 用户指定 or 密度自动（0/45/60） | 用户指定 or 0 | 0 | 同 bar | 同 bar |
| **legend_loc** | n_series≤5: inside; >5: outside | 同左 | 同左 | none | none |
| **y_min** | 用户指定 or 自动收紧 | 同左 | 同左 | 同左 | None |
| **marker_size** | — | n>50:3; n>20:4; 否则:5（直径） | `clamp(60/√n, 10, 80)`（面积） | — | — |
| **annot_fontsize** | — | — | — | — | n_cells>20: 6; 否则: font×0.85 |
| **x_max_ticks** | — | n>20: 10 | — | — | — |

> **Y轴自动收紧规则**：若数据最小值 > 最大值的 50%（数据远离 0），则自动将 Y 轴下限设为 `data_min - span×15%` 并取整到量级，否则保持 None（让 matplotlib 从 0 开始）。用户显式设置 `axes_y_min` 时，该自动计算被跳过。

> **图例移外规则**：当图例条目数 > 5 时，`legend_loc = "outside_right"`，同时 `figure_width` 额外加 1.2 英寸为图例留白。

> **刻度旋转规则**：计算"平均标签字符数 × 类别数 / (图幅宽度 × 8)"作为密度指标，> 1.4 → 60°，> 0.7 → 45°，否则 0°。

#### 调用方式

渲染器不需要直接实例化 `LayoutParams`，只需调用：

```python
from tools.layout import compute_layout

layout = compute_layout(df, spec, theme)
# layout.figure_width, layout.font_size, layout.legend_loc ... 等字段随即可用
```

**渲染流水线：**

```
render_plot(spec, data_source)
  ├─ _resolve_dataframe()     从缓存或文件路径取回 DataFrame
  ├─ apply_theme()            从 themes.py 取 ThemeConfig（静态风格）
  ├─ compute_layout()         从 layout.py 计算 LayoutParams（动态布局）
  └─ RENDERERS[chart_type]()  调用对应子渲染器
       ├─ _prepare_axes()        创建 Figure/Axes，figsize = layout.figure_width/height
       ├─ ... 绘制数据 ...
       ├─ _apply_axis_limits()   应用 Y 轴范围（来自 layout，已合并用户值与自动收紧）
       └─ _apply_theme_to_fig()  统一处理：字号/轴脊/网格/刻度旋转/图例位置
```

**`_apply_theme_to_fig()` 统一处理三件事**（子渲染器无需自行设置）：

1. **刻度旋转**：`plt.setp(xticklabels, rotation=layout.tick_rotation, ha=layout.tick_ha)`
2. **图例**：根据 `layout.legend_loc` 三态创建图例：
   - `"inside_best"` → `ax.legend(loc="best")`
   - `"outside_right"` → `ax.legend(bbox_to_anchor=(1.02, 1))`（图外右侧）
   - `"none"` → 不创建（heatmap / 单系列图的默认值）
3. **样式**：字号 / 轴脊可见性与线宽 / 网格 / 背景色（dark 主题）

**各子渲染器读取的关键 spec 字段：**

**bar 图：**
- `data_y` 为字符串时：单指标，支持 `data_group_by`、`data_error`、`params_sort`、`params_stacked`
- `data_y` 为列表时：自动 melt 宽表，每列名作为分组维度，`data_group_by` 被忽略
- 支持 `params_orientation`（横/纵）、`params_show_values`、`params_hatch` 系列

**line 图：**
- `data_y` 为字符串时：单条线，支持 `data_group_by`（多条线）、`data_error`（误差带）
- `data_y` 为列表时：每列对应一条折线，直接循环绘制
- 支持 `params_smooth`、`params_linestyle`、`params_line_colors`、`params_markers`

**scatter / box / heatmap：** 详见代码注释及 2.1 节字段表。

**heatmap 特殊说明：** `data_filter`（pandas query 过滤）仅在 heatmap 中已实现，其他图表类型暂未启用。

### 5.2 核心 API

**`PlotAgent.process_input(user_input)`**：主流水线，调用 LLM → 合并 delta → 校验 → 渲染。

**`PlotAgent.render_from_spec(spec_json)`**：新增方法，接受 JSON 字符串，**跳过 LLM 直接渲染**。
- 用于 UI 手动编辑 PlotSpec 后重新渲染
- 自动更新 `agent.current_spec`，下一轮 LLM 的 delta 以此为基础
- 依次执行：JSON 解析 → `fill_defaults` → `validate` → `render_plot`

**`PlotAgent.load_data(source)`**：加载 CSV，更新内部 `data_context` 和 `current_cache_key`。

**`PlotAgent.reset()`**：清空所有状态，开始新任务。

### 5.3 UI 功能说明

界面分三个区域：

| 区域 | 组件 | 功能 |
|------|------|------|
| 左列 | 文件上传 + 数据摘要 | 上传 CSV，展示 DataContext |

| 底部 | PlotSpec 编辑区 + 按钮行 | **可直接修改 JSON 并重新渲染** |

**PlotSpec 编辑区**（`gr.Code`，`interactive=True`）：用户可在此直接修改任意字段（如改标题、调坐标轴范围），点击「重新渲染」即时生效，不消耗 LLM token。

**按钮行：**

- 「重置」：清空所有状态

```python
def _layout_violin(df: pd.DataFrame, spec: dict, theme: ThemeConfig) -> LayoutParams:
    x_col = spec.get("data_x", "")
    y_col = spec.get("data_y", "")
    n_cat = int(df[x_col].nunique()) if x_col in df.columns else 4

    min_width = n_cat * 0.6 + 1.2
    fig_w = max(theme.figure_width, min_width)
    fig_h = fig_w * theme.aspect_ratio
    bar_width = max(0.1, min(0.6, 0.7 / max(n_cat, 1)))
    font_size = _scale_font(theme.font_size, fig_w, theme.figure_width)
    y_min, y_max = _compute_y_range(df, spec, y_col)
    tick_rotation, tick_ha = _auto_tick_rotation(
        list(df[x_col].unique()) if x_col in df.columns else [], n_cat, fig_w
    )
    return LayoutParams(
        figure_width=fig_w, figure_height=fig_h,
        font_size=font_size, legend_fontsize=round(font_size * 0.85, 1),
        tick_rotation=tick_rotation, tick_ha=tick_ha,
        legend_loc="none",
        y_min=y_min, y_max=y_max,
        bar_width=bar_width, marker_size=5.0,
        annot_fontsize=font_size, capsize=float(theme.line_width) * 2,
        x_max_ticks=None,
    )

_DISPATCHERS = {
    ...
    "violin": _layout_violin,   # ← 追加
}
```

```python
def _render_violin(
    df: pd.DataFrame, spec: dict, theme: ThemeConfig, layout: LayoutParams
) -> plt.Figure:
    fig, ax = _prepare_axes(spec, theme, layout)

        inner=spec.get("params_violin_inner", "box"),
        palette=theme.palette, linewidth=theme.line_width,
        width=layout.bar_width, ax=ax,
    )
    _apply_axis_limits(ax, layout)
    _apply_theme_to_fig(fig, ax, theme, layout)
    return fig

RENDERERS = {
```

#### 第二步：`tools/themes.py`

`ThemeConfig` 只包含静态风格字段，不含 `figure_height`（由 `aspect_ratio` 推导）和 `legend_fontsize`（由 LayoutEngine 推导）：

```python
THEMES: dict[str, ThemeConfig] = {
    ...
    "acl": ThemeConfig(
        font_family="Times New Roman",
        font_size=10,           # 基准字号（标准图幅下），实际字号由 LayoutEngine 缩放
        line_width=0.8,
        figure_width=6.0,       # 最小基准宽度（英寸），LayoutEngine 只扩大不缩小
        aspect_ratio=0.625,     # 高度 = 6.0 × 0.625 = 3.75 英寸
        dpi=300,
        spines=["left", "bottom"],
        grid=False,
        grid_style="--",
        legend_frameon=True,
        palette=["#2196F3", "#F44336", "#4CAF50", "#FF9800", "#9C27B0"],
    ),
}
```

| 期望图幅 | aspect_ratio |

| 5.0" × 3.75" | 0.75 |
| 6.0" × 4.0" | 0.667 |
| 6.0" × 3.75" | 0.625 |
